Summary

Overview

What this contains

  • Problem statement and approach
  • Data preparation and modelling workflow
  • Evaluation and interpretation (without overstating impact)
  • Documented outputs for review

Why it’s kept

This repository is maintained as a historical artefact. It shows early modelling foundations and reporting discipline, even if it is not part of the “featured” portfolio set.

Method

Approach (high level)

  1. Define the prediction task and evaluation approach (baseline vs model).
  2. Prepare data (types, missingness, feature selection/engineering as documented).
  3. Train the model and check performance on a hold-out set.
  4. Interpret outputs cautiously and document limitations.
